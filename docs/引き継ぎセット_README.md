## 3. 確定済み集計ルール（変更不可）

### 3-1. 売上
```
売上 = 案件金額（取得日ベース）- キャンセル受付日当日の案件金額
・再送（再送当否列=「再送」）は除外
・幸野有希子CRM は全体・サイト集計から除外（OP個人実績には反映）
・スタッフマスターを正として突合
```

### 3-2. キャンセル
```
通常キャンセル：キャンセル受付日が集計対象日のもの
　→ ダッシュボード・OP個人実績 両方に反映

考慮キャンセル：キャンセル受付日に「考慮」が含まれるもの
　（例：「2026/05/11 考慮」）
　→ OP個人実績にはキャンセルとして計上
　→ 全体・サイト別集計（ダッシュボード）からは除外
```

### 3-3. 人件費
```
時給制：時給 × 総労働時間（jinjer実績）

月給制（備考列=※月給）：月給 × 1.15 ÷ 当月営業日数 × 出勤日数

インセンティブ（月次）：月額 ÷ 当月営業日数 × 経過営業日数
　→ jinjer人件費に加算して合計人件費とする
```

### 3-4. 稼働単価【全箇所統一】
```
稼働単価 = 対象グループの売上合計 ÷ 対象グループの出勤日数合計
（重み付き平均）

※ダッシュボードKPI・詳細テーブル・ランク別・OP個人実績
　全箇所で同一計算式を使用すること
```

### 3-5. 原価率【全箇所統一】
```
原価率 = 人件費（jinjer + インセンティブ日割り）÷ 売上 × 100

※全体・サイト別・ランク別・OP個人実績 全箇所で同一計算式を使用
```

### 3-6. 着地見込み
```
着地見込み = 累計売上 ÷ 経過営業日数 × 当月営業日数
```

### 3-7. ランク別集計
```
ルーキー表示 = ルーキー + ルーキープラス を合算して「ルーキー」と表示
```

## 9. 実装チェックリスト（毎回確認）

```
□ 再送除外済みか
□ 幸野有希子CRM 除外済みか（ダッシュボード集計）
□ 考慮キャンセルの扱いは正しいか
□ B→D番号変換済みか
□ スタッフマスターと突合済みか
□ 同一指標は全ページで同一計算式か（稼働単価・原価率）
□ NaN・Infinity が含まれていないか
□ scriptブロックの波括弧が一致しているか（{の数 = }の数）
□ ヒートマップ更新済みか
□ インセンティブ日割りは経過営業日数で計算されているか
□ CHART_DATAは全営業日分の要素数になっているか
```

## 10. 計算方法統一ルール（重要）

```
ページをまたいで同一の指標が存在する場合、
実装前に計算式を明示・確認してから反映する。

複数箇所に同じ指標を追加・修正する際は、
全箇所で同一の計算式を使用することを必ず確認する。

ズレが生じた場合は根本の計算式レベルで統一し、
表示側の差異で誤魔化さない。
```

## 12. 過去のエラーと対処法（Lessons Learned）

| エラー | 原因 | 対処 |
|---|---|---|
| 白画面 | JSの構文エラー（波括弧不一致・NaN混入） | 波括弧数確認・fillna('')を必ず実施 |
| KPI数値が更新されない | 静的HTMLが残っていた | v28でPT_DATA一元管理に移行済み |
| チャートが1日分しか出ない | 2つのチャートが独立していた | CHART_DATA変数で統一済み |
| 稼働単価のズレ | ページ間で計算式が異なっていた | 全箇所で「売上合計÷出勤日数合計」に統一 |
| Google翻訳でフォルダ名が日本語化 | Chromeの自動翻訳 | GitHubでは翻訳をオフにする |
